# Region Utility Offices – Project Support

## Utilities within the Project Design Schedule - Overview

This overview outlines the critical integration of utility coordination throughout the project development process. Consistent with **WSDOT Utilities Manual Chapter 2**, early coordination is essential to mitigate project delays and cost overruns.

| Design Phase | Step | Task Description |
| :--- | :---: | :--- |
| **Scoping - 30%** | 1 | Research & Data Collection on Existing Utilities |
| **30% - 60%** | 2 | Utility Basemap and Utility Conflict Analysis |
| **~30% - 60%** | 3 | Utility Conflict Coordination Meetings |
| **~30% - 60%** | 4 | Utility Design and Construction Agreements |

---

### Key WSDOT Manual References

* **WSDOT Utilities Manual Section 2-1.02:** Emphasizes that utility identification must begin during the scoping phase to ensure utility impacts are considered in the project budget and schedule.
* **WSDOT Design Manual Section 300.04(3):** Notes that the Project Engineer is responsible for coordinating with the Region Utility Engineer to identify potential conflicts early in the design process (at or before 30% design).
* **WSDOT Utilities Manual Section 2-1.05:** Details the "Utility Conflict Matrix," which is the primary tool used during the **30% - 60%** phase for conflict analysis and resolution planning.
